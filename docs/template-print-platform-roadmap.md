# 模板打印平台远景规划

## 结论

`YiboLabel 0.x` 仍然继续聚焦标签打印系统。

`YiboLabel 1.x` 的远景方向可以升级为“模板打印平台”：把标签、出库单、送货单、入库单等固定版式打印需求统一到同一套本地模板、编辑、词库和输出体系里。

这个方向不是当前阶段的实现目标，而是后续架构演进时的产品边界参考。

## 当前阶段边界

### 0.x 继续做标签打印

`0.x` 的重点仍然是把当前标签打印工作流做稳：

- 标签模板库
- 标签画布编辑
- 文本、条码、二维码、图片、线条、矩形元素
- 词库辅助输入
- 本地模板保存、打开、复制、重命名、删除
- 标签机识别与打印
- 导入、导出、预览等日常可用性

这个阶段不应该为了表单需求提前引入复杂报表系统，也不应该让标签打印主线变重。

### 1.x 再考虑平台化

`1.x` 可以把产品从“标签打印工具”扩展为“固定版式模板打印工具”。

核心变化是：

- 标签只是文档类型之一
- 出库单等 A4 表单成为另一类文档
- 模板库、画布编辑、元素系统、词库系统尽量复用
- 打印链路按文档类型和目标设备分流

## 为什么这个方向成立

出库单这类需求与标签打印有相同的底层特征：

- 版式固定
- 主体结构长期不变
- 每次只改少量字段
- 需要快速编辑和打印
- 需要本地模板管理
- 不依赖云端和账号

因此它适合被 `YiboLabel` 的模板库和画布编辑体系承接。

同时，它与标签打印也有明显不同：

- 通常使用 A4 或其它普通纸张
- 通常走 Windows 系统打印机
- 可能有更多文本字段
- 可能有表格、边框、签字栏、备注区
- 打印预览和分页心智不同于标签

所以它不应该作为“标签打印机的一个特殊尺寸”硬塞进去，而应该作为新的文档类型进入系统。

## 产品定位

### 推荐定位

`YiboLabel 1.x` 推荐定位为：

本地优先的固定版式模板打印平台。

它解决的是：

- 设计固定版式模板
- 快速替换少量字段
- 借助词库减少重复输入
- 输出到合适的打印设备
- 保存本地模板，便于长期复用

### 不推荐定位

不建议把 `1.x` 直接定位为完整报表系统。

首轮平台化不应追求：

- 完整 Excel 模板兼容
- 自动分页报表
- 大批量数据驱动打印
- 多表关联业务数据
- 复杂数据库管理
- 多用户协作审批流

这些能力可以作为更远期方向，但不适合作为 `1.x` 起点。

## 文档类型

### 标签文档

标签文档继续服务当前核心场景。

特征：

- 小尺寸画布
- 面向标签纸
- 常见元素包括文本、条码、二维码、图片和简单图形
- 打印链路优先走标签机专用通道
- 需要关注浓度、间隙、方向、偏移等标签机参数

### 表单文档

表单文档服务出库单等固定版式纸张打印。

特征：

- A4、A5 或自定义纸张
- 面向系统打印机
- 常见元素包括文本、线条、矩形、图片、条码和二维码
- 打印链路优先走系统打印或 PDF 打印
- 需要关注纸张、边距、缩放、打印机选择和预览

第一版表单文档建议只支持单页固定版式。

## 打印链路

平台化后，打印不再等同于发送标签机指令。

应形成两类输出路径：

- 标签机专用打印：继续服务当前标签设备
- 系统打印：支持 Windows 下的普通打印机，包括 A4 喷墨打印机和激光打印机

系统打印可以优先基于 PDF 或系统打印对话框实现。

这样可以避免把 A4 表单强行转成标签机语义，也能让普通打印机自然参与进来。

## 词库方向

词库系统不需要一开始升级成完整结构化记录系统。

出库单字段虽然多，但并不总是彼此关联：

- 日期通常可以直接填当天
- 出库单号通常每张都不同
- 品名可以来自常用词库
- 规格可以来自常用词库
- 数量多数时候直接输入
- 备注可以来自常用词库

因此，`1.x` 仍可坚持弱结构化优先。

### 可保留的方式

- 对每个文本元素绑定一个或多个词库分组
- 输入时提供补全
- 通过内容浮窗点选候选
- 打印后可选收录常用值

### 可探索的增强

后续可以考虑“组合条目”。

例如：

- 规格 + 物料条码
- 品名 + 规格
- 客户名称 + 地址

组合条目可以帮助处理确实经常一起出现的字段，但不应该要求所有表单都以结构化记录方式录入。

## Excel 导入定位

不需要兼容 Excel 作为长期模板格式。

Excel 导入如果做，应该服务于减少重画成本，而不是承诺完整还原 Excel。

可接受的导入层级：

- 导入 Excel 转出的图片或 PDF 作为描图背景
- 读取简单文本和边框，生成表单初稿
- 从 Excel 数据中复制字段值到当前表单

不建议在 `1.x` 起点承诺：

- 高保真还原合并单元格
- 完整还原 Excel 打印区域
- 完整还原分页、页眉、页脚和缩放
- 以 `.xlsx` 作为主模板格式

如果已经接受在 `YiboLabel` 中一次性重画出库单，Excel 导入就只是辅助入口，不是核心依赖。

## 第一阶段平台化建议

如果进入 `1.x`，建议第一阶段只做固定单页表单。

建议范围：

- 新增文档类型：标签、表单
- 新增纸张预设：A4、A5、自定义
- 表单文档支持系统打印
- 表单文档支持 PDF 导出
- 复用现有元素系统
- 复用现有模板库
- 复用现有词库补全
- 支持把 Excel/PDF/图片作为重画参考背景

暂缓范围：

- 明细行自动扩展
- 自动分页
- 批量导入连续打印
- 复杂业务字段关联
- 完整 Excel 兼容
- 报表设计器级别功能

## 架构原则

### 文档类型先行

平台化时应先明确文档类型，而不是继续把所有东西都叫 `LabelDocument`。

可考虑抽象出更上层概念：

- `PrintDocument`
- `DocumentKind`
- `LabelDocument`
- `FormDocument`

具体命名以后实现时再定，但方向上应避免让标签语义继续外溢到表单场景。

### 共享编辑能力

标签和表单应该共享这些能力：

- 画布
- 元素拖拽、缩放、旋转
- 图层
- 对齐和微调
- 模板保存和打开
- 导入导出
- 词库辅助输入

### 分离输出能力

标签和表单应该分离这些能力：

- 打印机选择
- 打印参数
- 预览方式
- 输出渲染
- 实际发送到设备的链路

这样可以保持标签打印稳定，也给普通纸张打印留下清晰空间。

## 与现成方案的关系

这个方向存在替代方案。

### Excel 加脚本

优点：

- 上手快
- 可以继续用已有表格
- 成本低

短板：

- 模板管理弱
- 词库体验弱
- 长期维护容易分散
- 与标签系统割裂

### 报表工具

例如 FastReport、Stimulsoft 等。

优点：

- 报表、分页、数据绑定成熟
- 表格能力强

短板：

- 商业授权成本
- 集成重量大
- 对个人本地工具可能过度
- 与当前轻量标签工作流风格不一致

### 继续扩展 YiboLabel

优点：

- 标签和单据可以统一管理
- 本地优先
- 可按真实需求渐进实现
- 不需要引入重型平台

短板：

- 需要重构部分文档和打印抽象
- 系统打印链路需要补齐
- 表单能力需要谨慎控制范围

## 路线判断

当前最稳妥的路线是：

1. `0.x` 继续完成标签打印系统。
2. 在文档中保留 `1.x` 模板打印平台方向。
3. 等标签模板、词库、导出、打印稳定后，再引入文档类型。
4. `1.x` 首先支持单页固定表单和系统打印。
5. 更复杂的报表和批量数据能力继续后置。

一句话总结：

`YiboLabel 0.x` 是标签打印工具，`YiboLabel 1.x` 可以升级为本地固定版式模板打印平台。
