# 标题栏与工具栏临时改造方案

## 背景

YiboLabel 是本地优先的标签打印工具，不是浏览器式多标签软件。正常工作流通常只打开 1-2 个标签页，极端情况下也不应超过 4 个。现有三栏顶部结构分别承载标题/状态、工具栏、标签栏，视觉上形成“短、长、短”的节奏，既占用纵向空间，也让主要操作区产生偏移感。

本方案将顶部结构临时重构为两栏：

- 第一栏：标题栏，负责品牌、当前标签页、通知/状态、窗口控制。
- 第二栏：工具栏，负责导航、编辑历史、导入恢复、保存导出、打印相关操作。

目标是先稳定信息架构和空间分配，再进入代码改造。

## 设计目标

- 减少顶部纵向占用，取消独立标签栏。
- 让标签页符合打印软件的真实使用频率，而不是浏览器式多标签模型。
- 将同类功能聚合，减少“保存/导出/打印”等输出动作被拆散的问题。
- 让通知有固定槽位，不因内容长短推动其它元素或改变标题栏高度。
- 让打印机选择控件与其它工具栏按钮等高，避免独自撑高工具栏。
- 保留足够的窗口拖拽区域，窗口控制按钮保持稳定位置。

## 总体结构

```text
第一栏：标题栏
[YB] YiboLabel v0.7.41  [管箍21-44 · 已保存 ×] [+]    [通知/状态槽................]    [—][□][×]

第二栏：工具栏
[模板库] [词库] [文档与打印]   [撤消] [重做]   [导入为草稿] [恢复关闭]   [保存] [另存为模板] [导出]   [打印机: ● HB-Q2] [立即打印]
```

## 第一栏：标题栏

标题栏建议高度控制在 `48-52px`，内部使用单行横向布局，不再显示两行状态文字。

建议分区：

- 品牌区：`YB` 标识、`YiboLabel`、版本号。
- 标签区：当前打开的文档标签，显示文档名、保存状态、关闭按钮，并提供新建标签入口。
- 通知/状态槽：固定弹性区域，承载短通知或默认状态摘要。
- 窗口控制区：最小化、最大化/还原、关闭。

建议布局比例：

```text
品牌区：内容宽度，固定不被挤压
标签区：minmax(220px, 40vw)，最多显示 4 个标签
通知区：flex: 1，min-width: 180px，单行省略
窗口控制区：固定宽度
```

### 标签页规则

标签页放入标题栏后，不应设计成重浏览器标签，而应是紧凑的“工作文档切换器”。

建议样式：

- 当前标签使用高亮 pill 或浅底 tab。
- 已保存使用绿色点或小型状态 chip。
- 未保存使用橙色点，不额外拉高标题栏。
- 标签名过长时单行省略。
- 关闭按钮只在 hover 或当前标签中明显显示。
- 新建/恢复入口使用紧凑 `+` 按钮。

临时阶段可只支持 1-4 个标签平铺。若未来超过 4 个，再增加 `更多` 折叠菜单。

### 通知/状态槽规则

通知槽必须固定位置和高度，不参与自然宽度竞争。

显示策略：

- 无通知时：显示状态摘要，例如 `已保存 · HB-Q2 在线`。
- 普通通知：替换状态摘要，短时间后回到摘要。
- 重要通知：使用 warning/error 语义色，保留到用户下一次操作或明确关闭。
- 长通知：单行 `ellipsis`，完整内容放入 `title` 或 tooltip。
- 不允许通知推动窗口按钮、标签区或改变标题栏高度。

通知优先级：

1. 错误/打印失败。
2. 保存中/打印中/导出中。
3. 未保存修改。
4. 已保存/打印机在线。
5. 普通操作反馈。

## 第二栏：工具栏

工具栏建议高度统一为 `42-44px`，所有主按钮高度控制在 `32-36px`。

建议分组：

```text
导航组：
[模板库] [词库] [文档与打印]

编辑历史组：
[撤消] [重做]

入口恢复组：
[导入为草稿] [恢复关闭]

保存输出组：
[保存] [另存为模板] [导出]

打印组：
[打印机: ● HB-Q2] [立即打印]
```

布局规则：

- 同组按钮间距小，组与组之间间距大。
- `保存 / 另存为模板 / 导出 / 立即打印` 属于输出链路，应连续靠右。
- `撤消 / 重做` 必须成对出现，靠近左侧导航之后。
- `导入为草稿 / 恢复关闭` 属于入口与恢复链路，放在中间。
- 禁用态按钮保留位置，避免工具栏跳动。

## 打印机选择控件

打印机选择应与其它按钮同高，不再使用两行结构。

推荐形态：

```text
[● HB-Q2          v] [刷新]
```

或更紧凑：

```text
[打印机 · ● HB-Q2  v]
```

规则：

- 高度 `34-36px`。
- 固定宽度约 `200-240px`。
- 绿色点表示在线，橙色/灰色表示异常或离线。
- 详细状态放入 tooltip 或下拉内容。
- 刷新图标放在控件右侧，不额外增加高度。
- 与 `立即打印` 放在同一组，靠工具栏右侧。

## 响应式与溢出规则

虽然这是桌面软件，但窗口仍可能被缩窄，需要定义退化顺序。

建议折叠顺序：

1. 普通通知从完整文本变为省略文本。
2. 入口恢复组隐藏次要文字，仅保留图标和 tooltip。
3. `另存为模板` 可折叠到保存按钮的下拉菜单。
4. 标签区只保留当前标签，其他标签进入 `更多`。
5. 打印组不折叠到不可见，至少保留打印机状态和立即打印。

窗口控制区永远不参与折叠。

## 临时落地范围

第一阶段只改顶部结构，不触碰编辑器主体、打印逻辑、模板逻辑。

建议改动范围：

- `WorkspaceTopbar`：重组 DOM，将标签页能力并入标题栏。
- `EditorTabStrip`：改为标题栏内紧凑标签组件，或拆出可复用 `TitleTabs`。
- `App.css`：新增标题栏两栏布局、通知槽、工具栏分组、打印机 pill 样式。
- 保持现有事件处理函数和业务状态不变。

暂不处理：

- 标签超过 4 个后的复杂菜单。
- 命令面板。
- 全局快捷键提示重构。
- 主编辑画布和左右面板视觉重构。

## 验收标准

- 顶部区域从三栏变为两栏。
- 标题栏高度稳定，通知出现/消失不改变高度。
- 通知不会推动窗口控制按钮。
- 当前标签清晰可见，关闭/新建入口可用。
- `撤消 / 重做` 成对出现。
- `保存 / 另存为模板 / 导出 / 立即打印` 连续聚合。
- 打印机选择与其它工具栏按钮等高。
- 最大化窗口下顶部仍贴边，窗口拖拽区域可用。
- 常见 1-2 标签场景下，顶部视觉不再出现“短、长、短”的割裂感。

## 后续实现建议

建议下一轮按以下顺序落地：

1. 先重构 `WorkspaceTopbar` 的 JSX 结构，形成标题栏和工具栏两层。
2. 将 `EditorTabStrip` 的渲染逻辑迁入标题栏或拆成 `TitleTabStrip`。
3. 为通知槽建立固定布局规则，先复用当前 `status` 字符串。
4. 重做打印机选择控件为单行 pill。
5. 补齐撤消/重做按钮位置，并重新分组工具栏按钮。
6. 跑 `npm run build`、`npm run lint` 和桌面窗口冒烟检查。
