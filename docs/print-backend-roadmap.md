# 打印后端版本路线图

> 状态：当前演进路线。
> 当前用途：指导 `0.9.x` 到 `1.x` 的打印后端重构、系统打印接入和特殊标签机适配。

## 总方向

后端打印能力应从“当前标签机专用链路”调整为：

系统打印机主干 + 特殊打印机适配器。

这意味着：

- 系统打印机是默认通用模型，覆盖 Windows 队列、PDF/位图输出、普通纸张和未来 A4/A5 表单。
- 特殊打印机是独立适配器，覆盖 HB-Q2、Dlabel DLL、厂商 USB 通道和后续其它非标准标签机。
- 前端和模板不应长期直接依赖裸 `devicePath`，而应选择稳定的 `printTargetId`。
- 后端根据打印目标的 `channelKind` 路由到对应实现。

## 架构原则

### 系统打印优先

系统打印机应成为打印模型的主干。

原因：

- Windows 打印队列是普通打印机、虚拟 PDF 打印机、A4/A5 表单和后续平台化的共同入口。
- 系统打印更适合长期承接 `1.x` 的固定版式模板打印平台方向。
- 通用打印能力不应该继续被 HB-Q2 的厂商 DLL、USB 路径和 TSPL 细节绑住。

### 特殊打印隔离

HB-Q2 等特殊设备应作为适配器接入，而不是定义整个打印模型。

适配器需要负责：

- 发现厂商配置和 USB 设备路径。
- 探测厂商 DLL 是否存在、可加载、可初始化。
- 打开专用 USB 通道。
- 发送设备协议所需的 TSPL 或其它指令。
- 输出可诊断的失败原因。

### 选择目标而不是路径

文档层应从 `printerDevicePath` 逐步迁移到 `printTargetId`。

打印目标应包含：

- `id`：稳定目标标识。
- `displayName`：用户可读名称。
- `channelKind`：`windows-queue`、`vendor-usb` 等。
- `identity`：队列名、设备路径或适配器内部标识。
- `capabilities`：支持的纸张、DPI、输出格式和专用参数。
- `status`：在线、离线、不可用和诊断信息。

### 输出格式分离

打印不应只等于生成 TSPL。

需要逐步分出：

- `TsplOutput`：面向标签机专用通道。
- `BitmapOutput`：面向系统队列位图打印或部分标签机。
- `PdfOutput`：面向系统打印、PDF 导出和表单打印。

## 0.9.3 打印诊断版

目标：先看清楚机器上到底有哪些打印路径。

范围：

- 新增打印诊断 API。
- 枚举 Windows 系统打印机。
- 枚举 Dlabel/HB-Q2 厂商 USB 候选。
- 显示每条通道的状态、失败原因、DLL 路径、设备路径和队列名。
- 不大改打印 UI，只增强错误信息和诊断入口。

交付标准：

- 用户能区分“系统队列存在但不可用”和“厂商 DLL 缺失/加载失败/USB 打开失败”。
- 诊断结果能定位到具体阶段，而不是只显示“离线”。

## 0.9.4 打印目标模型重构

目标：把 `devicePath` 从核心打印模型里解耦。

范围：

- 引入 `PrintTarget` 或升级 `PrinterEndpoint`。
- 增加 `channelKind`：
  - `windows-queue`
  - `vendor-usb`
- 增加 `targetId`、`displayName`、`identity`、`capabilities`、`status`。
- 文档逐步从 `printerDevicePath` 迁移到 `printTargetId`。
- 保留旧模板兼容迁移。

交付标准：

- 后端能同时返回系统打印机和 HB-Q2 专用通道。
- 前端选择的是打印目标，而不是裸设备路径。
- 旧模板可以继续打开和打印。

## 0.9.5 系统打印基础版

目标：先让普通 Windows 打印机可用。

范围：

- 新增 `WindowsQueuePrinterChannel`。
- 支持系统打印测试页。
- 支持把当前标签渲染成位图后通过 Windows 打印队列输出。
- 支持选择系统默认打印机。
- HB-Q2 队列如果卡住，不把系统队列打印作为专用通道修复承诺。

交付标准：

- 普通喷墨、激光和虚拟 PDF 打印机可以走系统打印。
- HB-Q2 仍可单独走 `vendor-usb`。
- 系统打印失败与专用通道失败互不混淆。

## 0.9.6 HB-Q2 专用通道稳定版

目标：把当前这台标签机修稳。

范围：

- 将 `VendorUsbPrinterChannel` 独立成特殊打印机适配器。
- Dlabel DLL 路径支持自动探测和配置。
- probe 分级：
  - DLL 目录是否存在。
  - `DPrintCore.dll` 是否存在。
  - DLL 是否可加载。
  - factory 是否存在。
  - USB service 是否初始化。
  - `devicePath` 是否可打开。
  - payload 是否发送成功。
- 打印失败返回可读诊断。

交付标准：

- 失败时明确卡在哪一步。
- HB-Q2 专用通道和系统打印通道互不影响。
- 当前标签机可在专用通道下稳定测试打印和正式打印。

## 0.9.7 打印设置与校准迁移

目标：把校准从 `devicePath` 迁移到 `printTargetId`。

范围：

- 校准记录按 `printTargetId` 存储。
- 旧的 `devicePath` 校准自动迁移或兼容读取。
- 系统打印机和专用标签机拥有不同设置项。
- 标签机设置包括浓度、间隙、偏移和方向。
- 系统打印设置包括纸张、边距、缩放和方向。

交付标准：

- 切换打印目标后，显示对应类型的设置。
- 旧模板和旧校准不丢失。
- 打印检查能识别当前目标是否需要校准。

## 0.9.8 打印预览与输出统一

目标：统一渲染层，为 `1.0` 收口。

范围：

- 标签渲染输出分成 `TsplOutput`、`BitmapOutput`、`PdfOutput`。
- 系统打印优先使用位图或 PDF。
- HB-Q2 专用通道继续使用 TSPL。
- 打印检查页显示目标通道和输出格式。

交付标准：

- 同一份标签文档可以根据目标通道选择不同输出格式。
- 预览、导出和打印使用同一套渲染判断。

## 1.0 标签打印稳定版

目标：正式可长期使用的标签打印工具。

范围：

- 系统打印机可用。
- HB-Q2 专用通道可用。
- 打印诊断清晰。
- 模板、词库、校准、导出、备份恢复稳定。
- 不引入 A4 表单复杂能力。

交付标准：

- 当前标签打印工作流稳定闭环。
- 打印失败有明确修复建议。
- 系统打印主干和特殊打印适配器边界清楚。

## 1.1 系统打印增强版

目标：把普通系统打印做顺手。

范围：

- 系统打印预览更准确。
- 支持 PDF 打印和 PDF 导出路径。
- 支持更多纸张预设。
- 支持系统打印机默认设置记忆。
- 支持普通标签纸通过系统打印机输出。

## 1.2 文档类型基础版

目标：为模板打印平台铺路。

范围：

- 引入 `DocumentKind`：
  - `label`
  - `form`
- 不再让所有内容都只能表达为 `LabelDocument`。
- 表单文档第一版只支持单页固定版式。
- 标签和表单共用元素系统、模板库、词库和基础导出能力。

## 1.3 表单打印 MVP

目标：支持出库单/A4 单页模板。

范围：

- A4、A5 和自定义纸张。
- 系统打印作为唯一表单打印通道。
- PDF 导出。
- 图片/PDF 背景描图。
- 文本、线条、矩形、二维码和条码。

## 优先级判断

下一步应从 `0.9.3` 开始，不直接重写所有打印逻辑。

推荐顺序：

1. 诊断清楚。
2. 模型解耦。
3. 系统打印通道。
4. HB-Q2 专用通道稳定。
5. `1.0` 标签打印收口。

这样既不会继续被特殊标签机绑住，也不会为了未来 `1.x` 一次性重构过头。
