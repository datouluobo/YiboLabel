# 模板标签下放临时方案与实施计划

## 背景

当前模板编辑工作区已经把词库能力下放到左侧标签，左侧逐步成为编辑器内的主要工作台。

下一步需要把顶部“模板”入口也下放到左侧标签中，形成：

- 元素
- 词库
- 模板

这份文档用于固定临时方案和实施计划。当前只作为讨论后的落地依据，先不直接进入编码。

## 目标

模板标签的目标不是创建新的模板流程，而是承接原独立模板页的现有模板管理能力。

核心目标：

- 左侧统一承载模板、元素、词库三个编辑相关入口
- 模板标签用于管理当前已有模板
- 右侧临时占用为模板预览区
- 通过预览和标签切换减少页面突然变化
- 取消独立模板页后，仍保留模板复制、重命名、删除等管理能力

## 非目标

本轮不处理：

- 新建模板入口下放
- 导入 DDL 入口下放
- 模板高级筛选
- 模板批量操作
- 模板版本历史
- 复杂模板详情编辑

新建和导入仍保留在现有合适入口中，不放入左侧模板标签。

## 最终导航形态

顶部不再保留独立“模板”主入口。

模板编辑页左侧标签为：

- 元素
- 词库
- 模板

职责划分：

- 元素：编辑当前模板内容和元素列表
- 词库：管理词库分组/条目，并绑定或应用到元素
- 模板：管理已有模板，选择模板并在右侧预览

## 模板标签左侧方案

左侧模板标签定位为轻量管理后台。

### 固定区

固定区只保留：

- 搜索框

不放：

- 新建
- 导入 DDL
- 大段说明
- 模板详细信息

原因：

- 左侧宽度有限
- 模板标签的高频任务是查找已有模板
- 详细信息应放到右侧预览区承担

### 模板列表

列表项只显示简单信息：

- 模板名

可显示轻量状态：

- 当前正在编辑
- 已打开

列表行操作：

- 复制
- 重命名
- 删除

列表交互：

- 单击模板：选中模板，并在右侧显示预览
- 双击模板：打开或切换到该模板编辑
- 复制模板：复制后自动选中新模板并显示预览
- 重命名模板：保持当前选中和预览，只更新名称
- 删除模板：沿用现有确认逻辑

### 列表显示原则

- 只显示模板名，不显示尺寸、更新时间、元素数等详情
- 模板名超长时单行截断
- 行高与元素列表保持同类密度
- 行操作尽量与元素/词库列表的按钮风格一致

## 右侧模板预览区方案

当左侧切到“模板”标签时，右侧主区域临时进入模板预览模式。

### 预览区职责

右侧用于显示：

- 模板只读预览
- 模板详细信息
- 进入编辑的主操作

### 预览内容

建议显示：

- 模板名称
- 模板尺寸
- 元素数量
- 最近修改时间
- 关联打印机或纸张信息
- 只读画布预览

### 主操作

右侧只保留一个主操作：

- 编辑模板

如果模板已经打开，则文案可变为：

- 切到已打开模板

### 预览模式限制

预览模式不是编辑模式。

不显示：

- 元素选框
- 调整手柄
- 旋转手柄
- 词库拖放目标
- 内联编辑器
- 编辑标尺操作

可以保留：

- 缩放到合适视图
- 基础只读画布
- 模板元信息

## 标签切换交互规则

本方案的重点是减少右侧区域的突然变化。

### 从元素进入模板

当用户从“元素”标签切换到“模板”标签时：

1. 如果右侧当前打开画布来自一个已保存模板，则自动选中该模板，并在右侧显示该模板预览。
2. 如果当前画布是导入文件、草稿、未保存来源，或模板已经不存在，则右侧保持空白。
3. 不自动选中搜索结果第一项。
4. 不自动打开其它模板。

目的：

- 用户从正在编辑的模板进入模板管理时，应尽量看到同一个模板
- 如果无法推导目标，就不要突然跳到一个不相关模板

### 从模板进入元素

当用户从“模板”标签切换回“元素”标签时：

1. 如果当前正在预览一个模板，则打开或切换到该模板的编辑模式。
2. 如果该模板已经打开，则切到对应编辑标签。
3. 如果该模板尚未打开，则打开模板并进入编辑。
4. 如果当前没有预览模板，则恢复原来的当前编辑标签。
5. 如果没有任何编辑标签，则右侧保持空白。

目的：

- 用户在模板页确认了某个预览模板后，回到元素页应自然进入该模板编辑
- 减少“左侧切了标签，右侧突然回到另一个模板”的割裂感

### 从模板进入词库

建议规则：

1. 如果当前正在预览模板，不强制打开模板。
2. 词库仍围绕当前编辑上下文服务。
3. 如果没有当前编辑模板，右侧可保持空白。

原因：

- 词库是辅助编辑的管理面板
- 用户切词库不一定代表要编辑当前预览模板
- 避免过度隐式打开模板

## 状态设计

建议把模板预览状态从顶层页面模式中拆出来，变成编辑器内部状态。

建议状态：

```ts
sidebarTab: 'elements' | 'lexicon' | 'templates'
previewTemplateId: string | null
lastEditorTabId: string | null
```

派生显示：

```ts
mainPanelMode =
  sidebarTab === 'templates'
    ? 'template-preview'
    : 'editor'
```

### 切到模板标签

伪代码：

```ts
function switchToTemplatesTab() {
  setSidebarTab('templates')

  if (activeTab?.origin === 'template' && activeTab.templateId && templateExists(activeTab.templateId)) {
    setPreviewTemplateId(activeTab.templateId)
    return
  }

  setPreviewTemplateId(null)
}
```

### 切到元素标签

伪代码：

```ts
async function switchToElementsTab() {
  if (previewTemplateId) {
    await openOrFocusTemplate(previewTemplateId)
  } else {
    restoreLastEditorTabOrBlank()
  }

  setSidebarTab('elements')
}
```

### 切到词库标签

伪代码：

```ts
function switchToLexiconTab() {
  restoreLastEditorTabOrBlank()
  setSidebarTab('lexicon')
}
```

## 旧模板页面兼容

当前可能仍存在旧状态：

```ts
activeSurface === 'templates'
```

迁移规则：

- 启动恢复到旧模板页时，改为进入编辑器，并切到左侧“模板”标签
- 如果旧状态中能推导出选中模板，则设置 `previewTemplateId`
- 如果不能推导，则右侧保持空白

在新方案稳定后，再清理旧 `TemplateLibrary` 页面组件和相关 surface 分支。

## 删除与已打开模板的关系

删除模板需要避免数据丢失。

建议规则：

- 删除库中的模板，不应强行关闭已打开的编辑标签
- 如果被删除模板已有打开标签，打开标签应转为草稿或未关联模板状态
- 如果打开标签有未保存修改，必须保留编辑内容
- 删除后如果当前预览的就是该模板，则 `previewTemplateId` 置空，右侧显示空白

这部分需要在实现前检查现有模板删除逻辑是否已经处理。

## 复制与重命名规则

### 复制

复制后：

- 自动选中新复制模板
- 右侧显示新模板预览
- 不自动打开编辑

原因：

- 复制是管理动作，不等价于编辑动作
- 用户可以先确认复制结果，再决定是否进入编辑

### 重命名

重命名后：

- 保持当前选中模板
- 保持右侧预览
- 只更新左侧名称和右侧详情

## 空状态

### 模板列表为空

左侧：

- 显示“暂无模板”

右侧：

- 保持空白或显示极简空态
- 不提供新建/导入入口

### 搜索无结果

左侧：

- 显示“没有匹配模板”

右侧：

- 如果当前 `previewTemplateId` 仍存在，可继续显示当前预览
- 如果没有当前预览，则保持空白

### 无法推导预览目标

从元素进入模板，但当前画布不是已保存模板时：

- 左侧不自动选中模板
- 右侧保持空白

## 视觉原则

左侧模板列表：

- 密度接近元素列表
- 只显示模板名
- 行操作与元素/词库列表按钮一致
- 避免卡片化、避免多行元信息

右侧预览区：

- 可以比编辑画布更安静
- 信息详情集中在预览旁或预览下方
- 主按钮明确且唯一
- 空白状态不要喧宾夺主

## 实施计划

## 第一阶段：状态和路由设计落地

目标：

- 建立左侧三标签中的“模板”标签
- 增加模板预览状态
- 为旧模板页状态提供兼容回落

任务：

- 扩展左侧标签枚举为 `elements | lexicon | templates`
- 新增 `previewTemplateId`
- 梳理 `activeSurface === 'templates'` 的兼容路径
- 明确 `模板 -> 元素` 的打开或切换逻辑

交付结果：

- 状态边界清楚
- 旧页面入口不会造成断链

## 第二阶段：左侧模板列表

目标：

- 在左侧模板标签中管理已有模板

任务：

- 加入模板搜索
- 渲染模板名列表
- 支持单击选中
- 支持双击打开
- 接入复制、重命名、删除
- 显示已打开/当前编辑的轻量状态

交付结果：

- 左侧模板标签具备完整基础管理能力

## 第三阶段：右侧模板预览区

目标：

- 用右侧主区域承载只读模板预览和详情

任务：

- 新增 `TemplatePreviewPanel`
- 复用现有画布预览能力
- 禁用编辑交互
- 展示模板详情
- 增加“编辑模板/切到已打开模板”主按钮

交付结果：

- 用户可在不打开编辑 tab 的情况下确认模板内容

## 第四阶段：跨标签过渡

目标：

- 减少页面切换的突然变化

任务：

- 从元素进入模板时，自动选中当前已保存模板
- 无法推导时右侧保持空白
- 从模板进入元素时，打开或切换当前预览模板
- 从模板进入词库时，不强制打开预览模板

交付结果：

- 标签切换符合用户当前关注对象

## 第五阶段：顶部入口收束与旧组件清理

目标：

- 完成模板入口下放

任务：

- 移除顶部模板入口
- 保留旧模板页代码一轮作为回退
- 稳定后清理旧 `TemplateLibrary` 和相关 surface 分支

交付结果：

- 模板入口统一在左侧标签中
- 顶部工具栏更轻

## 验收清单

实施后至少检查：

1. 左侧是否只显示模板名和必要行操作
2. 搜索是否只影响模板列表
3. 单击模板是否只预览、不打开
4. 双击模板是否打开或切换编辑
5. 从元素进入模板时，已保存模板是否能自动选中自身
6. 从元素进入模板且无法推导模板时，右侧是否保持空白
7. 从模板进入元素时，是否进入当前预览模板编辑
8. 删除当前预览模板后，右侧是否安全置空
9. 删除已打开模板时，是否不丢失打开标签中的未保存内容
10. 顶部是否不再出现独立模板入口

## 风险与控制

### 风险 1：隐式打开模板造成困惑

从模板切回元素会打开当前预览模板，这是本方案的关键交互。

控制方式：

- 右侧主按钮文案明确使用“编辑模板”
- 切回元素时只对当前预览模板生效
- 没有预览模板时不自动打开其它模板

### 风险 2：左侧模板标签变得过重

如果左侧加入太多详情，窄栏会难用。

控制方式：

- 左侧只显示模板名
- 详情全部放右侧
- 新建和导入不进入模板标签

### 风险 3：旧模板页状态残留

旧 `activeSurface === 'templates'` 可能仍存在于保存的工作区状态。

控制方式：

- 启动时兼容回落
- 稳定后再删除旧 surface 分支

### 风险 4：删除模板影响已打开编辑

删除库模板时，不能丢掉正在编辑的内容。

控制方式：

- 删除库模板和关闭编辑标签分离
- 已打开标签保留为草稿或未关联模板
- 删除前沿用确认逻辑

## 最终结论

模板入口可以下放到左侧标签，但模板标签应承担轻量管理后台职责。

最终结构：

- 左侧 `模板` 标签管理已有模板
- 左侧只显示搜索、模板名列表、复制/重命名/删除
- 右侧临时显示模板预览和详情
- 从元素进模板时尽量预览当前已保存模板
- 从模板回元素时进入当前预览模板编辑
- 无法推导目标时右侧保持空白

该方案优先保证对象连续性，减少页面突变，同时保留取消独立模板页后的管理能力。
