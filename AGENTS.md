# 项目约定

## 删除安全规则

- 允许删除已确认的构建产物、诊断产物和其它明确无用的中间文件。
- 删除前必须先列出计划删除的具体路径，并得到用户明确确认。
- 未经确认，不要直接删除文件或目录。
- 执行删除时，优先使用明确路径，避免模糊匹配。

## 仓库操作路由

- 执行 Git 或 GitHub 仓库操作时，优先遵循全局 skill `git-solo-workflow`；单人仓库默认直接走主分支流程，只有在大改动、试验性改动或用户明确要求时才使用临时分支或 PR。

## 新项目头脑风暴

- 每次开启新项目，在前期头脑风暴阶段，首先考虑是否已有现成产品可直接使用或改造。
- 如有现成产品候选，需要主动提出来，并列出它们的费用、使用限制、缺少的关键功能或其它明显短板。

## 版本管理

- 每次对项目进行修改并交付新构建后，必须同步将项目版本号递增 `+0.0.1`。
- 版本号至少同时更新这些位置：`Directory.Build.props`、`src/YiboLabel.App/ClientApp/package.json`、`src/YiboLabel.App/ClientApp/package-lock.json`。
